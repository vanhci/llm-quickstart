# 想学大模型应用开发，找教程太麻烦了，于是自己写了一个

---

朋友问我："想基于 GPT 或者 DeepSeek 做一个自己的 AI 应用，要从哪里开始？"

这个问题我问过自己很多遍。

说实话，市面上的教程真的太多了——视频课、付费专栏、开源项目，堆起来能砸死人。但真正适合中国开发者、入门即能跑通、代码还能复用的教程，我翻了很久，没找到。

要么是只讲概念，没有代码。要么是代码有了，但跑不通（API Key 申请不到、数据不知道怎么准备、版本早就过时了）。要么是太"学院派"了，一上来就讲 Transformer 架构……

所以我决定自己写一个。

不是什么高深的研究成果，就是一个**从零搞定 LLM 应用开发**的实战教程。分两个阶段，代码开源，链接在文末。

---

## 🤔 这个教程怎么学的？

我把学习路径分成**两个阶段**：

- **阶段一：快速入门**（8天）。建立完整概念地图，跑通 Prompt、RAG、Agent、向量数据库、LangChain、API 和 Docker Demo。目标是从"听过 LLM"到"能独立做出一个可用的 AI 应用"。
- **阶段二：独立开发能力**（4个模块）。补齐工程化 Agent、进阶 RAG、评测、安全、部署和真实项目交付能力。目标是从 Demo 走向真实业务系统。

不用纠结数学，不用死磕理论，直接上手写代码。

---

## 📅 阶段一：快速入门

### Day 1 · 提示词工程 🧠

很多人觉得"不就是写 Prompt 吗？有什么好学的？"

真不是。我见过太多人拿着 GPT 输出废话连篇的回答，然后吐槽模型不够智能——问题出在 Prompt 上。

Day 1 我们会讲：

- **结构化 Prompt 的写法**（角色设定 + 任务描述 + 输出格式）
- **Few-shot 示例注入**（让模型照着例子学）
- **思维链（Chain of Thought）激活**（复杂推理任务必备）
- **温度、Top-P 这些参数到底怎么调**

学完这一天，你会发现自己以前写的 Prompt，90% 都是白写。

### Day 2 · RAG 核心 📄

GPT 不知道你公司内部的数据，不知道你的文档内容。

RAG（检索增强生成）就是为了解决这个问题：**给模型外接一个知识库，让它只在相关文档里找答案。**

Day 2 我们手把手教你：

- 如何加载 PDF、Markdown、TXT 等各种文档
- 怎么做**中文文本切分**（这坑很多人踩过）
- Embedding 模型怎么选、怎么调
- 如何搭建一个**可用的检索流程**

实战任务：做一个**你自己的 AI 知识库问答工具**，接公司文档、笔记、论文都能用。

### Day 3 · Agent 与工具调用 🤖

光会回答问题还不够，AI 得能**做事**。

Agent 就是让大模型自己决定：要用什么工具、怎么调用、下一步做什么。

Day 3 我们会实现：

- **Function Calling / ReAct 范式**（推理与行动结合）
- 给 Agent 挂载**搜索、计算、文件读写**等工具
- 多轮对话中的**记忆管理**（让 AI 记住上下文）
- 如何设计一个能帮你**自动完成复杂任务**的 Agent

实战任务：做一个能自动搜索信息、整理报告、发送邮件的**私人 AI 助理**。

### Day 4 · 向量数据库 🗄️

RAG 效果好不好的关键之一，在于向量数据库选型。

Day 4 我们讲清楚：

- 向量检索的原理（Embedding、相似度计算……）
- **ChromaDB 基础操作**（本地开发首选）
- 向量索引优化思路（HNSW 等算法）
- 如何处理**海量文档下的检索延迟**问题

### Day 5 · 生产级实践 📊

Demo 跑通了，但上线还差得远。

Day 5 我们补充：

- **日志记录**：请求追踪、错误定位
- **成本控制**：Token 计算、用量监控
- **简单评估**：输出质量主观打分 + 回归测试思路

### Day 6 · 模型替代方案 🔄

OpenAI API 在国内用不了？这一章覆盖：

- **DeepSeek**（¥1/百万 token，便宜）
- **Groq**（免费，速度快）
- **Ollama**（本地免费，完全离线）
- **MiniMax**（国内直连，支付方便）

统一封装，切换只改 1 行代码。

### Day 7 · LangChain 快速入门 🔗

手写 RAG、Agent 链路代码很长？LangChain 把这些封装好了，**代码量减少 70%**。

只讲最核心的 3 个组件：

- **LLMChain**：Prompt + Model 封装
- **RetrievalQA**：RAG 专用链
- **ConversationalAgent**：带记忆的对话助手

学框架是为了快速搭原型验证思路，不是为了依赖它。

### Day 8 · 完整项目实战 🚀

把前面所有"零件"组装成**可部署的私有知识库系统**。

从 0 到 1 完成：

- 文档读取 → 分块 → Embedding → 向量库
- 检索 + 生成（流式输出）
- **FastAPI API 服务**
- **Docker 一键部署**

学完这章，你的前端、客户端就能调用你的专属 AI 服务了。

## 📅 阶段二：独立开发能力

学完阶段一，你已经能跑 Demo 了。阶段二的目标是**让它能上生产**。

### Module 1 · Agent 工程化

- 从简单工具调用升级为**可靠 Agent 状态机**
- 处理工具调用失败、重试、权限控制
- **人工确认节点**（关键操作需人审批）

### Module 2 · RAG 进阶

- **召回质量提升**：重排（ReRank）、混合检索
- **答案可信度**：引用来源追踪、置信度标注
- **知识库可维护性**：增量更新、版本管理

### Module 3 · 评测与测试

- 建立**评测集**，做 RAG 和 Agent 回归测试
- 量化评估：召回率、答案准确率、响应时间
- 用自动化测试保障每次迭代不出错

### Module 4 · 安全与部署

- **Prompt Injection** 防护、敏感信息过滤
- 生产服务：日志、监控、配置管理
- **Docker 容器化部署**，依赖锁定
