# 我想学 LLM 应用开发，就让 AI 给我写了一套教程

---

我想系统学一下 LLM 应用开发。

不是那种"了解一下"的学习，是真的能独立做项目、能搭 RAG、能写 Agent、能部署上线的那种。

但我懒得看长篇大论的文档，也不想被付费专栏割韭菜。

于是我想到一个办法：**让 AI 帮我写教程来教我自己学。**

---

## 怎么让 AI 写出真正能用的教程？

一开始我尝试直接问 GPT："给我写一个 LLM 开发教程。"

它输出了几千字，结构清晰，概念完备——然后我看了一下代码示例，发现跑不通。API 版本早换了，依赖包名字都不对。

问题出在哪？

**AI 不知道我需要什么。** 它只是在堆概念，不是在帮我建立能力。

所以我换了个思路：**不是让 AI 替我写答案，而是让它帮我设计学习路径，然后我来验证每一步是否真的能跑通。**

具体做法是：

![人机协作迭代闭环](https://excalidraw.com/#json=pKfR8XyrqEdUEBIm5Ph3S,EQ6twqlw0ovpXkuIyfAMnw)

1. 我告诉 AI 我会什么、我想做什么、我的约束条件是什么
2. AI 设计一套学习路径，每一步都附带可运行的代码
3. 我跑代码，验证它说的是不是真的
4. 跑不通就反馈给 AI，让它修正
5. 重复，直到整条路径跑通

这个过程迭代了很多轮，最终形成的，就是现在这套教程。

## 阶段一：8天，建立完整的开发地图

![LLM应用开发·8天学习路线图](https://excalidraw.com/#json=lS3cMMHd4RxksnSlUHttj,L9V7sOQroNQK4kcJdIjs2g)

### 第1天：Prompt工程

很多人觉得 Prompt 有什么好学的。

但我见过最常见的失败案例就是：同一个工具，新手写的 Prompt 输出废话连篇，老手写的 Prompt 一句话解决战斗。

差别在哪里？**结构**。角色设定、任务描述、输出格式、Few-shot示例、CoT思维链——这些不需要什么高级框架，只需要知道怎么组织语言。

学完这天，你写的 Prompt 会比之前精准 10 倍。

### 第2天：RAG核心

RAG 是把大模型落地的核心技术，没有之一。

它的逻辑很简单：**模型不是百科全书，但可以是学霸——你给它资料，它来回答。**

这天的目标：让你理解 Embedding 的本质、向量检索的原理，以及如何从 0 搭建一个知识库问答系统。

做出来的效果：上传一份 PDF，直接用自然语言问答。

### 第3天：Agent与工具调用

Agent 是让 AI 从"回答问题"升级到"替你做事"的关键。

比如：让 AI 自动帮你查天气、整理搜索结果、发一封邮件——而不只是生成一段文字。

这天的核心是 **ReAct 范式**：让模型自己决定调用什么工具、怎么组合步骤、遇到错误怎么办。

做出来的效果：一个能自动完成多步骤任务的 AI 助手。

### 第4天：向量数据库

很多人跳过这个，觉得"用现成的就行"。

但如果你不知道向量检索的原理，遇到召回质量差、响应慢的时候，你根本不知道怎么调。

这天的目标：**让你在理解原理的基础上玩转 ChromaDB**，而不是只会调 API。

### 第5-6天：生产级实践 & 模型替代

Demo 跑通了，但上线还差得远。

- 第5天：日志怎么记、成本怎么控、效果怎么评估
- 第6天：OpenAI 用不了怎么办——DeepSeek、Ollama、Groq、MiniMax，一个接口自由切换

### 第7-8天：LangChain + 完整项目实战

手写 RAG 链路代码很长？LangChain 可以帮你**减少 70% 的代码量**。

最后两天，从 0 到 1 完成一个**可部署的私有知识库系统**：文档读取 → 分块 → Embedding → 向量库 → 检索生成 → FastAPI 接口 → Docker 部署。

## 阶段二：让它能上生产

学完阶段一，你已经能跑 Demo 了。但 Demo 距离真正服务用户，还差几步：

- Agent 调用工具失败了怎么办？需要重试吗？需要人工确认吗？
- RAG 的答案总是差一点，怎么提升召回质量？怎么加引用来源？
- 怎么评估 AI 的回答质量？用什么测试集做回归测试？
- Prompt Injection 是什么？怎么防止用户注入恶意指令？

阶段二的四个模块，就是为了解决这些问题。
