# 我花了两周时间，写了一份"不废话"的 LLM 开发教程

---

上周五晚上 11 点，我又被问住了。

朋友发来消息："我想给公司的知识库接个 AI 助手，让员工可以用自然语言查制度、查流程，但不懂技术，有没有简单的方法？"

我想了想，发现这个问题根本没有标准答案。

市面上能搜到的教程，要么是"5天学会AI"的短视频收割流量，要么是"基于LangChain构建企业级知识库问答系统"的付费专栏——价格不便宜，代码还跑不通。

我试过很多种方案：**RAG是什么、Embedding怎么做、要不要上向量数据库、LangChain生态那么复杂从哪入手**……光是把这些概念拼起来形成一个完整的开发路径，就已经劝退了一半的人。

所以我决定自己写一份。

不是学术论文，不追概念完整，只追一个目标：**让你用最短的时间，从"装不上依赖"到"跑通一个真实可用的AI应用"**。

---

## 第一个月：我以为 Prompt 就是全部

我第一次接触大模型应用开发，是去年这时候。

当时觉得很简单——写个 Prompt 调用 GPT API，不就行了吗？

结果现实给我上了一课：同一个 Prompt，第一遍回答得不错，第三遍就开始"幻觉"满天飞。问它公司内部的报销流程，它一本正经地胡说八道。

我意识到：**模型不知道我不知道的东西**，它只会很自信地编答案。

这就是 RAG（检索增强生成）存在的意义——不是让模型"记住"更多知识，而是给它一个"查资料"的能力。

---

## 那个让我失眠的技术问题：私有知识库怎么做？

让我真正下定决心系统学这个方向的，是一个真实的业务场景：

**我要给一个几百页的产品手册接一个问答机器人。**

用户用自然语言提问，AI 从手册里找到相关内容，直接给出答案。不需要员工去翻 PDF、去搜索、去Ctrl+F。

当时我以为这很简单——把文档扔给模型不就行了？

后来才知道：

- 文档太长，一次性塞给模型会超上下文限制
- 语义搜索不是关键词匹配，需要 Embedding 和向量数据库
- 分块（chunking）策略直接影响召回质量——分太大查不准，分太小答案不完整

**我踩了整整三周的坑**，才从 0 到 1 搭出了一个能用的版本。

这个过程里积累的经验，就成了这份教程的核心内容。

## 阶段一：8天，建立完整的开发地图

### 第1天：Prompt工程

很多人觉得 Prompt 有什么好学的。

但我见过最常见的失败案例就是：同一个工具，新手写的 Prompt 输出废话连篇，老手写的 Prompt 一句话解决战斗。

差别在哪里？**结构**。角色设定、任务描述、输出格式、Few-shot示例、CoT思维链——这些不需要你会什么高级框架，只需要知道怎么组织语言。

学完这天，你写的 Prompt 会比之前精准 10 倍。

### 第2天：RAG核心

RAG 是把大模型落地的核心技术，没有之一。

它的逻辑很简单：**模型不是百科全书，但可以是学霸——你给它资料，它来回答。**

这天的目标：让你理解 Embedding 的本质、向量检索的原理，以及如何从 0 搭建一个知识库问答系统。

做出来的效果：上传一份 PDF，直接用自然语言问答。

### 第3天：Agent与工具调用

Agent 是让 AI 从"回答问题"升级到"替你做事"的关键。

比如：让 AI 自动帮你查今天的天气、整理搜索结果、写入文件、发一封邮件——而不只是生成一段文字。

这天的核心是 **ReAct 范式**：让模型自己决定调用什么工具、怎么组合步骤、遇到错误怎么办。

做出来的效果：一个能自动完成多步骤任务的 AI 助手。

### 第4天：向量数据库

很多人跳过这个，觉得"用现成的就行"。

但如果你不知道向量检索的原理，遇到召回质量差、响应慢的时候，你根本不知道怎么调。

这天的目标：**让你在理解原理的基础上玩转 ChromaDB**，而不是只会调 API。

### 第5-6天：生产级实践 & 模型替代

Demo 跑通了，但上线还差得远。

- 第5天：日志怎么记、成本怎么控、效果怎么评估
- 第6天：OpenAI 用不了怎么办——DeepSeek、Ollama、Groq、MiniMax，一个接口自由切换

### 第7-8天：LangChain + 完整项目实战

手写 RAG 链路代码很长？LangChain 可以帮你**减少 70% 的代码量**。

最后两天，从 0 到 1 完成一个**可部署的私有知识库系统**：文档读取 → 分块 → Embedding → 向量库 → 检索生成 → FastAPI 接口 → Docker 部署。

做出来的效果：你的前端或客户端可以直接调用这个 AI 服务。

## 阶段二：让它能上生产

学完阶段一，你已经能跑 Demo 了。但 Demo 距离真正服务用户，还差几步：

- Agent 调用工具失败了怎么办？需要重试吗？需要人工确认吗？
- RAG 的答案总是差一点，怎么提升召回质量？怎么加引用来源？
- 怎么评估 AI 的回答质量？用什么测试集做回归测试？
- Prompt Injection 是什么？怎么防止用户注入恶意指令？

阶段二的四个模块，就是为了解决这些问题。
