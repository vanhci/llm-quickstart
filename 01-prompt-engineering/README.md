# Day 1 · 提示词工程 —— 让模型听懂你的话

## 什么是 Prompt Engineering？

Prompt Engineering（提示词工程）就是**学会怎么跟 AI 说话**。

同样的模型，会提问的人得到满分答案，不会提问的人得到废话。Prompt 工程就是研究怎么组织语言，让模型输出更准确、更符合预期的回答。

> 通俗理解：你是在"指挥"一个读过整个互联网但有点死板的助理——你描述得越清楚，它做得越到位。

---

## 三大核心原则

### 1. 清晰具体
不要只说"帮我写文案"，要说"帮我写一条朋友圈文案，推广我的螺蛳粉，要接地气，带emoji，100字以内"。

### 2. 给例子
用 Few-shot（少样本）提示——直接给模型看 1-2 个示例，它会照着模仿。

### 3. 分解任务
复杂问题拆成小步骤，让模型一步步来（Chain of Thought）。

---

## 常用高级技巧

### Chain of Thought（思维链）
引导模型一步步思考，而不是直接给出答案。

```
不要问："这个数学题答案是多少？"
而是问："请分步骤解答：先...然后...最后..."
```

### 角色扮演
让模型扮演专家角色，立刻提升回答质量。

```
你是一个有10年经验的电商客服，风格亲切专业。
用户问题：[用户问题]
```

### Few-shot（少样本）
在 Prompt 里给模型 1-3 个示例，它会照着格式和风格模仿。

---

## 工作流程

```
用户输入
   ↓
编写 Prompt（清晰 + 具体 + 给例子）
   ↓
模型理解指令
   ↓
执行推理（CoT / 角色 / Few-shot）
   ↓
输出回答
```
