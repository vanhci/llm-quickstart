# Module 2 · RAG 进阶

目标：把基础知识库问答升级为可维护、可评估、可信赖的 RAG 系统。

## 基础 RAG 的常见问题

- 文档解析后内容很脏。
- chunk 太大或太小，召回不稳定。
- 用户问法和文档表达不一致。
- 检索结果相关但不能回答问题。
- 答案没有引用来源。
- 知识库更新后旧数据重复。
- 不同用户看到不该看的文档。

## 进阶 RAG 流程

```text
文档上传
  ↓
解析与清洗
  ↓
按结构分块
  ↓
生成 metadata
  ↓
Embedding 入库
  ↓
查询改写
  ↓
混合检索
  ↓
重排 rerank
  ↓
上下文压缩
  ↓
带引用生成答案
  ↓
答案可信度评估
```

## 必须掌握的能力

### 1. 文档解析与清洗

需要处理：

- 页眉页脚。
- 表格。
- PDF 换行。
- 重复内容。
- 空白段落。
- 编码问题。

### 2. 分块策略

常见策略：

- 固定长度分块。
- 按标题层级分块。
- 按 Markdown 结构分块。
- 按语义分块。
- 父子 chunk：小块召回，大块生成。

### 3. Metadata 设计

建议至少记录：

- `doc_id`
- `source`
- `title`
- `page`
- `section`
- `created_at`
- `owner`
- `permission_group`

### 4. 检索质量提升

可逐步加入：

- query rewrite 查询改写。
- keyword + vector 混合检索。
- rerank 重排。
- 多路召回。
- 相似度阈值过滤。
- 上下文去重。

### 5. 答案引用

答案应该能说明依据：

- 引用文档名。
- 引用页码或章节。
- 引用片段。
- 无依据时明确说没有找到。

## 推荐练习

1. 给 `08-complete-project` 的文档入库增加 metadata。
2. 问答接口返回 `sources`，包含文档名和片段内容。
3. 加入相似度阈值，低于阈值时拒答。
4. 实现 Markdown 标题分块。
5. 构造 20 个问题，记录每个问题是否召回正确文档。

## 验收标准

完成本模块后，你应该能实现：

- 文档可增量入库，不重复写入。
- 答案带引用来源。
- 检索结果有相似度和 metadata。
- 能用固定问题集评估召回质量。
- 能解释 chunk 大小、top_k、阈值变化对结果的影响。
