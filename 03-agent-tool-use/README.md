# Day 3 · Agent 与工具调用 —— 让模型帮你执行操作

## 什么是 Agent？

**Agent（智能体）** 就是给模型"手脚"——让它不只能回答问题，还能执行操作。

> 想象你有一个实习生：你告诉它目标，它自己思考、查资料、做决定、执行任务，最后给你结果。
> 这就是 Agent。

---

## 为什么需要 Agent？

- **查实时信息**：模型训练数据有截止日期，不知道今天天气、股价、新闻
- **执行操作**：发邮件、写文件、调用 API、操作数据库
- **多步推理**：复杂任务需要多轮思考和行动
- **个性化**：记住用户偏好，提供定制化服务

---

## ReAct 模式

ReAct = **Re**asoning + **Act**ing

让模型循环：思考 → 行动 → 观察结果 → 再思考

```
用户：帮我查一下北京今天天气

🤔 思考：我需要调用天气工具
🎬 行动：调用 get_weather(city="北京")
👀 观察：返回 "晴，26度，东南风3级"
🤔 思考：天气信息已获取，可以回答用户了
✅ 回答：北京今天晴朗，气温26度，适宜出行
```

---

## 工具调用流程

```
用户问题
   ↓
模型判断是否需要工具
   ↓
选择工具（search/calculator/code...）
   ↓
执行工具，获取结果
   ↓
把结果放回上下文
   ↓
继续推理或直接回答
```

---

## 多步骤任务规划（实战）

真实场景中的 Agent 需要完成**复杂多步骤任务**，比如：
> "帮我查英伟达营收，算增长比例，发邮件给我"

这种任务需要：
1. 先搜索 → 获取数据
2. 再计算 → 处理数据
3. 最后执行 → 发送结果

Agent 的核心能力就是**自主规划 + 逐步执行**。

---

## 系统提示词工程（高级）

想让 Agent 保持一致的角色和行为？靠的是**系统提示词**。

```python
SYSTEM_PROMPT = """你是一个叫小明的保险顾问。

【角色设定】
- 专业、耐心、亲切
- 用通俗语言解释保险条款
- 不推销，实事求是

【行为规则】
1. 回答风格与角色一致
2. 记住用户透露的偏好
3. 不知道就说不知道，不编造
4. 适当体现记忆（但不要每次重复）"""
```
