# Day 2 · RAG 核心 —— 让模型"读懂"你的知识库

## 什么是 RAG？

**RAG = Retrieval-Augmented Generation（检索增强生成）**

通俗理解：就是给模型一张"开卷考试的资料"，让它根据资料回答问题，而不是凭空编造。

> 想象你要参加一场法律考试，监考老师允许你带一堆法律条文进考场，但你得自己翻书找答案——RAG 就是帮模型"翻书"的系统。

---

## 为什么需要 RAG？

1. **私有知识**：模型不知道你公司的产品、你的笔记、内部文档
2. **避免幻觉**：模型会"编造"答案，给它资料就能减少胡说八道
3. **实时更新**：知识库更新后，模型立刻能查到，不需要重新训练

---

## RAG 工作流程

```
用户问题
   ↓
Embedding 模型（把文字变成向量）
   ↓
向量数据库检索（找最相关的 N 条）
   ↓
把检索结果注入 Prompt
   ↓
大模型生成回答
   ↓
回答用户
```

```
用户问题 → "我的订单什么时候发货？"
     ↓
Embedding → [0.23, -0.45, 0.78, ...]（向量）
     ↓
向量数据库检索 → Top-3 相关文档
     ↓
Prompt = 用户问题 + 检索到的文档
     ↓
模型回答
```

---

## 核心概念：Embedding

**Embedding（嵌入）** 就是把文字变成一串数字（向量）。

相似的内容，数字也"差不多"。比如：
- "苹果水果" 和 "橙子营养" → 向量接近
- "苹果水果" 和 "iPhone 手机" → 向量较远

这就是为什么能用"相似度"找到相关内容。
